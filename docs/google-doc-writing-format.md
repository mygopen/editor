# MyGoPen 查核報告 Google 文件撰文格式

這份格式給編輯群撰寫 Google 文件時使用。只要保留欄位標題，並把內容填入 `[寫這邊]`，系統就能穩定轉成 Blogger HTML。

## 核心原則

- 請保留欄位標題文字，例如 `導言摘要段：`、`原始謠傳版本：`、`查證解釋：`。
- 編輯只替換 `[寫這邊]`，不要刪除或改名欄位標題。
- 圖片請直接插入 Google 文件，不要只貼外部圖片網址。
- 連結請用 Google 文件的超連結功能，系統會轉成 `<a href="...">文字</a>`。
- 需要紅字強調的內容，請在 Google 文件中設為紅色；系統會轉成 `<b><span style="color: red;">...</span></b>`。
- `結論：` 請獨立成一行，系統會轉成 `<h3 style="text-align: left;">結論</h3>`。

## 目前採用的欄位式文件結構

```text
你可以先知道：
[寫這邊]

導言摘要段：[寫這邊]

首圖：[img]大標：
[寫這邊]

原始謠傳版本：[寫這邊]

主要流傳這個影像：[img]/[video]

並在社群平台流傳：[img]

查證解釋：[寫這邊]

資料來源：[寫這邊]

延伸閱讀：[寫這邊]
```

## 各欄位填寫規則

- `你可以先知道：` 下方的 `[寫這邊]`：填查核重點，可寫成（1）（2）（3）條列，也可寫成單段說明。
- `導言摘要段：[寫這邊]`：填文章開頭導言，會轉成 `<div class="intro_words">...</div>`。
- `首圖：[img]大標：`：`[img]` 代表可在附近插入首圖；`[寫這邊]` 填文章主標，會轉成 `<h2>...</h2>`。
- `原始謠傳版本：[寫這邊]`：填謠言本體，會轉成 `<blockquote class="tr_bq">...</blockquote>`。
- `主要流傳這個影像：[img]/[video]`：可在這個欄位下方插入影片截圖、代表圖，或填影片說明。
- `並在社群平台流傳：[img]`：可在這個欄位下方插入社群截圖。
- `查證解釋：[寫這邊]`：填完整查證內容，會轉成 `<blockquote class="yestrue">...</blockquote>`。
- `資料來源：[寫這邊]`：填資料單位與連結。連結請用 Google 文件超連結功能。
- `延伸閱讀：[寫這邊]`：填延伸文章連結；沒有延伸閱讀時可留空。

## 對應 Blogger HTML

| Google 文件欄位 | Blogger HTML |
| --- | --- |
| `你可以先知道：` | `<div class="quote_style">...</div>` |
| `導言摘要段：` | `<div class="intro_words">...</div>` |
| `首圖：[img]` | Blogger 首圖區，系統會先輸出圖片 placeholder |
| `首圖：[img]大標：` 下方 `[寫這邊]` | `<h2>大標的謠言</h2>` |
| `原始謠傳版本：` | `<blockquote class="tr_bq">謠言本體</blockquote>` |
| `主要流傳這個影像：` | 一般說明文字與影片/圖片位置 |
| `並在社群平台流傳：` | 社群截圖位置 |
| `查證解釋：` | `<blockquote class="yestrue">...</blockquote>` |
| `資料來源：` | `資料來源：<br />...` |
| `延伸閱讀：` | `延伸閱讀：<br />...` |

## 圖片處理

轉換時，系統會把 Google 文件中的內嵌圖片另存到：

```text
exported-images/Google文件ID/image-01.jpg
exported-images/Google文件ID/image-02.png
```

編輯可從這個資料夾手動取圖，上傳到 Google Blogger，取得 `blogger.googleusercontent.com` 圖片 URL 後，再貼回系統的「圖片替換」欄位，重新產生 HTML。

## 最小可用範例

```text
你可以先知道：
（1）網傳說法缺乏根據。
（2）專家指出，傳言將 A 與 B 混為一談。

導言摘要段：網傳「某某說法」的影片訊息，聲稱......。經查證，......。

首圖：[img]大標：
網傳某某說法是真的？

原始謠傳版本：這是謠言本體第一段。這是謠言本體第二段。

主要流傳這個影像：[img]/[video]

並在社群平台流傳：[img]

查證解釋：網傳訊息原始出處為何？
MyGoPen 以「關鍵字」搜尋，可以找到......。

結論：
總結來說，網傳說法不正確。

資料來源：官方單位 - 資料標題（請加超連結）
```
